# bigint-mul-bench

多倍長整数の乗算アルゴリズムをRustで自前実装し、桁数ごとの速度を比較するプロジェクトです。

RubyKaigi 2026のMatzキーノートで [Spinel](https://github.com/matz/spinel)（Ruby AOTコンパイラ）のBigInt乗算に触れられたのをきっかけに作りました。

## 実装アルゴリズム

| アルゴリズム | 計算量 | ファイル | 概要 |
|:---|:---|:---|:---|
| 筆算法（Schoolbook） | O(n²) | `src/schoolbook.rs` | 各桁の全ペアを掛ける素朴な方法 |
| からつば法（Karatsuba） | O(n^1.585) | `src/karatsuba.rs` | 半分に分割して乗算を4回→3回に削減 |
| Toom-3 | O(n^1.465) | `src/toom3.rs` | 3分割+多項式補間で乗算を9回→5回に削減 |

外部のBigIntライブラリは使わず、すべてゼロから実装しています。

## ファイル構成

```
src/
├── common.rs      — 共通ユーティリティ（BASE, 加減算, シフト等）
├── schoolbook.rs  — 筆算法
├── karatsuba.rs   — からつば法
├── toom3.rs       — Toom-3
├── lib.rs         — モジュール定義とテスト
└── main.rs        — ベンチマーク実行
```

## ベンチマーク結果

`cargo run --release` で簡易ベンチマークを実行できます。

```
  digits |     schoolbook |      karatsuba |          toom3 |  K/S ratio |  T/S ratio
----------------------------------------------------------------------------------------
      10 |         137 ns |         170 ns |         173 ns |      0.81x |      0.80x
      50 |         4.1 µs |         5.8 µs |         6.8 µs |      0.70x |      0.60x
     100 |        27.8 µs |        17.7 µs |        15.6 µs |      1.57x |      1.78x
     200 |        71.6 µs |        39.3 µs |        40.1 µs |      1.82x |      1.78x
     500 |       396.7 µs |       146.4 µs |       145.9 µs |      2.71x |      2.72x
    1000 |        1.58 ms |       430.8 µs |       438.7 µs |      3.67x |      3.60x
    2000 |        6.44 ms |        1.34 ms |        1.33 ms |      4.80x |      4.85x
    5000 |       41.50 ms |        6.48 ms |        5.91 ms |      6.41x |      7.02x
```

桁数が増えるほどアルゴリズムの差が顕著になり、5000桁ではからつば法が筆算の約6.4倍、Toom-3が約7倍速くなっています。

## 使い方

```sh
# 簡易ベンチマーク
cargo run --release

# テスト
cargo test

# criterion によるベンチマーク
cargo bench
```
