# bigint-mul-bench

多倍長整数の乗算アルゴリズムをRustで自前実装し、桁数ごとの速度を比較するプロジェクトです。

RubyKaigi 2026のMatzキーノートで [Spinel](https://github.com/matz/spinel)（Ruby AOTコンパイラ）のBigInt乗算に触れられたのをきっかけに作りました。

## 実装アルゴリズム

| アルゴリズム | 計算量 | 概要 |
|:---|:---|:---|
| 筆算法（Schoolbook） | O(n²) | 各桁の全ペアを掛ける素朴な方法 |
| からつば法（Karatsuba） | O(n^1.585) | 半分に分割して乗算を4回→3回に削減 |
| Toom-3 | O(n^1.465) | 3分割+多項式補間で乗算を9回→5回に削減 |

外部のBigIntライブラリは使わず、すべてゼロから実装しています。

## ベンチマーク結果

`cargo run --release` で簡易ベンチマークを実行できます。

```
  digits |     schoolbook |      karatsuba |          toom3 |  K/S ratio |  T/S ratio
----------------------------------------------------------------------------------------
      10 |         330 ns |         394 ns |         392 ns |      0.84x |      0.84x
      50 |        12.2 µs |         9.2 µs |         4.6 µs |      1.33x |      2.64x
     100 |        24.0 µs |        16.1 µs |        12.6 µs |      1.49x |      1.91x
     200 |        68.2 µs |        36.4 µs |        36.6 µs |      1.87x |      1.86x
     500 |       397.8 µs |       136.8 µs |       133.3 µs |      2.91x |      3.15x
    1000 |        1.58 ms |       429.8 µs |       414.8 µs |      3.65x |      3.82x
    2000 |        6.48 ms |        1.29 ms |        1.25 ms |      5.04x |      5.20x
    5000 |       41.00 ms |        6.09 ms |        5.36 ms |      6.74x |      7.65x
```

桁数が増えるほどアルゴリズムの差が顕著になり、5000桁ではからつば法が筆算の約7倍、Toom-3が約7.6倍速くなっています。

## 使い方

```sh
# 簡易ベンチマーク
cargo run --release

# テスト
cargo test

# criterion によるベンチマーク
cargo bench
```
